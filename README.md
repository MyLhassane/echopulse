# 📡 EchoPulse (005.echopulse)

> **"لا ننتظر الترند، بل نراقب ولادته."**

**صدى التقنية (EchoPulse)** هو نظام أرشفة واستكشاف مؤتمت بالكامل (Agentic Workflow). يعمل كرادار صامت يمسح مستودعات GitHub يومياً لاقتناص المشاريع التقنية والبرمجيات الصاعدة التي ولدت خلال الأيام السبعة الماضية، قبل أن تضج بها المواقع الإخبارية.

---

## ✨ الميزات الرئيسية (Core Features)

- **رصد موضوعي من المنبع:** بدلاً من الفلترة العمياء، يعتمد النظام على استخلاص الكلمات المفتاحية (Topics) التي يضعها المطورون أنفسهم لمشاريعهم، مما يضمن دقة التخصص.
- **تصفية حية (Live Tag Filtering):** واجهة التقرير اليومي مزودة بمحرك جافا سكريبت مدمج (Client-side) يسمح للزائر بالضغط على أي وسم (Tag) لتصفية المشاريع فورياً دون الحاجة لإعادة تحميل الصفحة.
- **بوابة عبور مركزية (Root Portal):** يولد النظام تلقائياً صفحة هبوط (Landing Page) في جذر المشروع، تعمل كفهرس ديناميكي يُحدّث ذاتياً ليربط الزوار بكافة التقارير الزمنية السابقة.
- **أتمتة سحابية 100%:** يعمل النظام باستقلالية تامة عبر GitHub Actions (Cron Job) في منتصف الليل (UTC)، ليقوم بالجلب، التوليد، والنشر دون أي تدخل بشري.

---

## 🏗️ هيكلية الأرشفة (The Chrono-Archive)

تم تصميم بنية المجلدات لإرضاء أقصى معايير التنظيم، حيث يتم حفظ البيانات الخام (للمحللين) والواجهات البصرية (للقراء) في شجرة زمنية دقيقة:

```text
005.echopulse/
├── index.html          <-- (بوابة الهبوط المركزية والفهرس)
└── archive/
    └── YEAR/
        └── MONTH/
            └── DAY/
                ├── raw_data.json   <-- (البيانات الخام المهيكلة للتحليل المستقبلي)
                └── index.html      <-- (التقرير اليومي البصري التفاعلي)
````

---

## ⚙️ تقنيات العمل (The Stack)

- **المحرك الأساسي:** Python 3 (للتنقيب ومعالجة البيانات) 🐍
    
- **التشغيل الذاتي:** GitHub Actions (CI/CD & Cron) 🤖
    
- **الواجهة الأمامية:** Tailwind CSS (للتصميم) + Vanilla JS (للفلترة الفورية) ⚡
    
- **مصدر البيانات:** GitHub REST API v3 🛡️
    

---

## 🚀 رحلة تشغيل النظام (Workflow Lifecycle)

1. **الاستيقاظ:** يفتح GitHub سيرفر افتراضي يومياً الساعة 00:00.
    
2. **الاستكشاف:** يبحث `main.py` عن أسرع المستودعات نمواً (في آخر 7 أيام).
    
3. **البناء:** يتم توليد ملف `raw_data.json` وبناء بطاقات HTML تفاعلية للتقرير اليومي.
    
4. **التحديث:** يقوم السكريبت بمسح الأرشيف وتحديث روابط البوابة الرئيسية (Root Index).
    
5. **الرفع:** يتم عمل `Commit` و `Push` تلقائياً لتظهر النتائج على صفحات GitHub Pages.
